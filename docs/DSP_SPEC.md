# DSP Specification (v0.3.0)

This is a design overview of the deterministic DSP layer in
`axonos-pipeline-core`. The **normative**, bit-exact arithmetic lives in
[`PIPELINE_CONTRACT.md`](PIPELINE_CONTRACT.md) В§9; what is and is not asserted
lives in [`CLAIMS.md`](CLAIMS.md). This document does not restate those вЂ” it
explains the shape of the layer and points at the pinned definitions.

> This is an **engineering demonstrator**, not a medical device and not a
> validated clinical signal chain. See [`SAFETY_NOTES.md`](SAFETY_NOTES.md).

## 1. Architecture

```text
RawFrame -> Epoch -> (DSP) -> FeatureVector -> ClassifierDecision
                       |                              |
            DC removal / FIR / IIR filter bank   kernel boundary
            (this document)                      (only boundary-safe type)
```

The DSP stages are in-pipeline transforms. Their outputs are ordinary sample
buffers and are **not** boundary-safe: only the terminal `ClassifierDecision`
may cross the application boundary ([`PRIVACY_BOUNDARY.md`](PRIVACY_BOUNDARY.md)).

## 2. Sample model

- Samples are `i32` carrying 24-bit ADC counts, sign-extended
  (`ADC24_MIN = в€’8_388_608`, `ADC24_MAX = 8_388_607`).
- DSP functions operate **per channel** on a contiguous buffer; de-interleaving
  a channel out of a `RawFrame` is the acquisition layer's responsibility.
- Sample rate is a validated `SampleRate` newtype (non-zero by construction;
  250 / 500 / 1000 Hz are first-class).

## 3. Fixed-point model

All deterministic DSP is **integer fixed-point** вЂ” no floating point ever runs
on the data path, so results are bit-exact across platforms and against the
conformance vectors. The shared rules (PIPELINE_CONTRACT В§9):

- accumulation in `i64`;
- coefficients in `Q15` for the IIR sections (`BIQUAD_ONE = 1 << 15`);
- a round-half-up bias of `1 << 14` before the arithmetic right shift;
- **saturation** into the `i32` range on every output (no wrapping, no panic);
- division (mean removal) truncated toward zero.

## 4. Stages

### 4.1 Stateless primitives (`dsp`)
- `remove_mean` вЂ” integer DC (mean) removal (PIPELINE_CONTRACT В§9.1).
- `fir` вЂ” causal fixed-point FIR convolution engine, caller-owned coefficients,
  **no** frequency-response claim (В§9.2).

### 4.2 Stateful IIR filter bank (`filter`) вЂ” new in v0.3.0
Single-channel, allocation-free filters with `step` / `process` / `reset` /
`state_hash`. A multi-channel caller owns one filter per channel (`[Biquad; C]`).

- **DC blocker** вЂ” first-order high-pass `y[n] = x[n] в€’ x[nв€’1] + RВ·y[nв€’1]`,
  `R` a `Q15` pole (default `0.995`). `with_r` rejects `R в€‰ (0,1)` with
  `InvalidCoefficient`. (В§9.3)
- **Notch** (`NotchMode`) вЂ” `Hz50` / `Hz60` / `Disabled`, a `Q15` Direct-Form-I
  biquad. Coefficients are **tabulated** for 250 / 500 / 1000 Hz; any other rate
  is rejected with `UnsupportedSampleRate`. (В§9.4)
- **Band-pass** (`BandpassPreset`) вЂ” `MotorIntent` (в‰€ 8вЂ“30 Hz), `Attention`
  (в‰€ 4вЂ“12 Hz), `SafetyWide` (в‰€ 1вЂ“40 Hz), `Disabled`; same biquad and rate
  handling. (В§9.4)

Coefficients were computed **offline** (RBJ cookbook) and are stored as integer
`Q15` constants; the crate never designs a filter at runtime, so the `no_std`
core needs no floating point. The band edges are **engineering choices for a
demonstrator**, each realised as a single second-order section вЂ” there is no
certified frequency response and no clinical validation ([`CLAIMS.md`](CLAIMS.md)).

## 5. Artifact detection

`artifact_scan` screens a block against a positive amplitude threshold (in ADC
counts) and reports `Clean` / `AmplitudeExceeded` / `Saturated` (saturation
dominates). Threshold-to-microvolt conversion is the acquisition layer's job.
A richer multi-flag detector (flatline / spike / dropout) is future work.

## 6. Feature extraction

Not yet implemented as a deterministic stage. `FeatureVector<D>` is an `f32`
placeholder; the fixed-point feature path (RMS, abs-mean, zero-crossings, вЂ¦) is
roadmap **v0.4.0** and is **not** part of any conformance claim until then
([`LIMITATIONS.md`](LIMITATIONS.md)).

## 7. State hash

Every stateful filter exposes `state_hash() -> u64`: a deterministic FNV-1a 64
digest over its coefficients and state, each field as little-endian `i32` вЂ” no
pointers or addresses, so it is stable across platforms and runs. The exact
byte order is normative in PIPELINE_CONTRACT В§9.3вЂ“В§9.4 and pinned by the
`biquad` / `dc_blocker` vectors.

## 8. Replay vectors

DSP behaviour is pinned by language-neutral vectors in
[`../vectors/pipeline-vectors-v0.3.0.json`](../vectors/pipeline-vectors-v0.3.0.json),
generated by `tools/gen_test_vectors.py` (the single source of truth) and
exercised from `tests/conformance.rs`. The filter sections (`biquad`,
`dc_blocker`) run each design / the DC blocker over a shared `filter_signal` and
pin both the output **and** the post-run `state_hash`. See the **atomic-update
rule** in [`../vectors/README.md`](../vectors/README.md).

## 9. Limitations

- Single second-order sections only; no cascaded / higher-order designs, no
  certified frequency response, no clinical validation.
- No fixed-point feature extraction yet (v0.4.0); the artifact detector is the
  simple block screen above.
- Nothing below the acquisition frame is modelled (no AFE, ADC driver, or
  transport). See [`LIMITATIONS.md`](LIMITATIONS.md).
