## 1. Scope

The contract covers the **signal â†’ decision** path: from a raw acquisition
frame to a pipeline-terminal `ClassifierDecision`. It does **not** cover the

## 2. Sample model

- A sample is a signed integer **ADC count**, not a voltage.
- The reference acquisition front-end is 24-bit; counts are sign-extended into
  `i32`. The representable range is therefore
  `ADC24_MIN = -8_388_608 ..= ADC24_MAX = 8_388_607`.
- A `RawFrame` stores samples **time-major, interleaved, column-compacted**:
  for an enabled-channel count `n`, the value for time index `t` and storage
  column `col` is at linear index `t * n + col`.
- **Column compaction**: enabled channels occupy storage columns
  `0 .. n` in ascending channel-index order. `ChannelMask::column_of(ch)`
  returns the storage column of channel `ch`; `channel_at(col)` is its inverse.

A frame is well-formed iff its channel mask has at least one enabled channel
and the sample length is a non-zero multiple of the enabled-channel count.

## 3. Frame checksum (normative)

`RawFrame::checksum()` is **FNV-1a, 64-bit** over the following byte sequence,
in this exact order, each integer **little-endian**:

1. `seq`               â€” `u32`
2. `timestamp_us`      â€” `u64`
3. `sample_rate_hz`    â€” `u32`
4. `channel_mask`      â€” `u16`
5. `samples_per_channel` â€” `u32`
6. each sample, in storage order â€” `i32`

FNV-1a constants: offset basis `0xcbf29ce484222325`, prime `0x100000001b3`.
FNV-1a is a non-cryptographic integrity check and is **never** used for
authentication. Anchor digests and the fixture-frame digest are pinned in the
conformance vectors (`PV-FNV-*`, `PV-FRAME-001`).

## 4. Artifact screening (normative)

`artifact_scan(samples, threshold_counts)`:

- `threshold_counts` **must be positive** (ADC counts); a non-positive
  threshold is the error `InvalidThreshold`.
- A sample at or beyond a converter rail
  (`>= ADC24_MAX` or `<= ADC24_MIN`) yields `Saturated`.
- Otherwise a sample with magnitude strictly greater than `threshold_counts`
  yields `AmplitudeExceeded`.
- **Saturation dominates**: if any sample saturates, the result is `Saturated`
  regardless of other samples.
- A frame with no flagged sample is `Clean`.

**Counts vs microvolts.** The AxonOS Standard's acquisition default expresses
an artifact threshold of Â±120 ÂµV. Conversion from microvolts to counts is an
**acquisition-layer** responsibility and is intentionally *not* performed in
`pipeline-core`. For a 24-bit converter with analog front-end gain `G` and
reference voltage `VREF` (in ÂµV):

```text
counts = round( microvolts * G * 2^23 / VREF )
```

The acquisition layer supplies `threshold_counts` already converted. This
keeps `pipeline-core` free of floating-point analog assumptions and exactly
reproducible.

## 5. Windowing (normative)

For `samples_per_channel = N`, window length `W`, and hop `H`:

```text
window_count = (N - W) / H + 1          (integer division)
```

with errors:

- `InvalidWindow`  if `W == 0` or `H == 0`;
- `WindowTooLarge` if `W > N`.

`RawFrame::epochs(W, H)` yields exactly `window_count` epochs; epoch `k`
(`0`-based) begins at frame time index `k * H` and has length `W`. The
iterator is `ExactSizeIterator` and its reported length equals `window_count`.
Cases are pinned by `PV-WIN-*`.

## 6. Application boundary (normative)

Only types implementing the **sealed** trait `BoundarySafe` may cross the
application boundary. In v0.1.0 the sole implementor is `ClassifierDecision`.
`RawFrame`, `Epoch`, and `FeatureVector` are intentionally **not**
`BoundarySafe` and cannot be made so downstream (the trait is sealed). See
[`PRIVACY_BOUNDARY.md`](PRIVACY_BOUNDARY.md).

`ClassifierDecision` is either `NoIntent` or
`Intent { class: u8, confidence_permille: u16 }` with
`confidence_permille` in `0 ..= 1000`; an out-of-range confidence is the error
`InvalidConfidence`.

## 7. Conformance and the atomic-update rule

An implementation conforms to v0.1.0 iff it reproduces every vector in
`vectors/pipeline-vectors-v0.1.0.json`. The vectors are generated by
`tools/gen_test_vectors.py` (the single source of truth) and verified by
`tools/validate_vectors.py`.

Any change to a vector or fixture **must** re-run the generator and commit all
its outputs â€” including a regenerated `vectors/SHA256SUMS` â€” in the same
commit. This is enforced in CI.

## 8. Versioning

The vector set carries its own `vector_version`, independent of the crate
version. A crate release need not bump the vector version if behaviour is
unchanged; a behavioural change bumps both as appropriate.
