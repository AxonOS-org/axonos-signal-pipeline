# AxonOS Signal Pipeline — Contract (v0.6.0)

This document is the normative reference for the v0.6.0 type contract. The
Rust crate `axonos-pipeline-core` is an implementation of it; the JSON
conformance vectors are its language-neutral test surface. Where this document
and the vectors agree, that agreement is the specification.

## 1. Scope

The contract covers the **signal → decision** path: from a raw acquisition
frame to a pipeline-terminal `ClassifierDecision`. It does **not** cover the
conversion to the canonical `IntentObservation` wire type (RFC-0006 §4), which
is performed by the kernel under consent gating, nor any analog or transport
concern below the acquisition frame.

## 2. Sample model

- A sample is a signed integer **ADC count**, not a voltage.
- The reference acquisition front-end is 24-bit; counts are sign-extended into
  `i32`. The representable range is therefore
  `ADC24_MIN = -8_388_608 ..= ADC24_MAX = 8_388_607`.
- A `RawFrame` stores samples **time-major, interleaved, column-compacted**:
  for an enabled-channel count `n`, the value for time index `t` and storage
  column `col` is at linear index `t * n + col`.
- **Column compaction**: enabled channels occupy storage columns
  `0 .. n` in ascending channel-index order. `ChannelMask::column_of(ch)`
  returns the storage column of channel `ch`; `channel_at(col)` is its inverse.

A frame is well-formed iff its channel mask has at least one enabled channel
and the sample length is a non-zero multiple of the enabled-channel count.

## 3. Frame checksum (normative)

`RawFrame::checksum()` is **FNV-1a, 64-bit** over the following byte sequence,
in this exact order, each integer **little-endian**:

1. `seq`               — `u32`
2. `timestamp_us`      — `u64`
3. `sample_rate_hz`    — `u32`
4. `channel_mask`      — `u16`
5. `samples_per_channel` — `u32`
6. each sample, in storage order — `i32`

FNV-1a constants: offset basis `0xcbf29ce484222325`, prime `0x100000001b3`.
FNV-1a is a non-cryptographic integrity check and is **never** used for
authentication. Anchor digests and the fixture-frame digest are pinned in the
conformance vectors (`PV-FNV-*`, `PV-FRAME-001`).

## 4. Artifact screening (normative)

`artifact_scan(samples, threshold_counts)`:

- `threshold_counts` **must be positive** (ADC counts); a non-positive
  threshold is the error `InvalidThreshold`.
- A sample at or beyond a converter rail
  (`>= ADC24_MAX` or `<= ADC24_MIN`) yields `Saturated`.
- Otherwise a sample with magnitude strictly greater than `threshold_counts`
  yields `AmplitudeExceeded`.
- **Saturation dominates**: if any sample saturates, the result is `Saturated`
  regardless of other samples.
- A frame with no flagged sample is `Clean`.

**Counts vs microvolts.** The AxonOS Standard's acquisition default expresses
an artifact threshold of ±120 µV. Conversion from microvolts to counts is an
**acquisition-layer** responsibility and is intentionally *not* performed in
`pipeline-core`. For a 24-bit converter with analog front-end gain `G` and
reference voltage `VREF` (in µV):

```text
counts = round( microvolts * G * 2^23 / VREF )
```

The acquisition layer supplies `threshold_counts` already converted. This
keeps `pipeline-core` free of floating-point analog assumptions and exactly
reproducible.

## 5. Windowing (normative)

For `samples_per_channel = N`, window length `W`, and hop `H`:

```text
window_count = (N - W) / H + 1          (integer division)
```

with errors:

- `InvalidWindow`  if `W == 0` or `H == 0`;
- `WindowTooLarge` if `W > N`.

`RawFrame::epochs(W, H)` yields exactly `window_count` epochs; epoch `k`
(`0`-based) begins at frame time index `k * H` and has length `W`. The
iterator is `ExactSizeIterator` and its reported length equals `window_count`.
Cases are pinned by `PV-WIN-*`.

## 6. Application boundary (normative)

Only types implementing the **sealed** trait `BoundarySafe` may cross the
application boundary. In v0.6.0 the sole implementor is `ClassifierDecision`.
`RawFrame`, `Epoch`, and `FeatureVector` are intentionally **not**
`BoundarySafe` and cannot be made so downstream (the trait is sealed). See
[`PRIVACY_BOUNDARY.md`](PRIVACY_BOUNDARY.md).

`ClassifierDecision` is either `NoIntent` or
`Intent { class: u8, confidence_permille: u16 }` with
`confidence_permille` in `0 ..= 1000`; an out-of-range confidence is the error
`InvalidConfidence`.

## 7. Conformance and the atomic-update rule

An implementation conforms to v0.6.0 iff it reproduces every vector in
`vectors/pipeline-vectors-v0.6.0.json`. The vectors are generated by
`tools/gen_test_vectors.py` (the single source of truth) and verified by
`tools/validate_vectors.py`.

Any change to a vector or fixture **must** re-run the generator and commit all
its outputs — including a regenerated `vectors/SHA256SUMS` — in the same
commit. This is enforced in CI.

## 8. Versioning

The vector set carries its own `vector_version`, independent of the crate
version. A crate release need not bump the vector version if behaviour is
unchanged; a behavioural change bumps both as appropriate.

## 9. DSP primitives (normative)

All DSP is **integer fixed-point**; floating point is never used on the DSP
data path, so results are bit-exact across platforms and against the
conformance vectors. Each function operates per channel on a contiguous sample
buffer; the caller supplies an output buffer of equal length.

### 9.1 DC (mean) removal

`remove_mean(input, out)` writes `input[i] - mean` to `out[i]` and returns
`mean`, where `mean = sum(input) / input.len()` with the division **truncated
toward zero** (matching Rust's `i64 /`; e.g. `sum = -7, len = 2 → mean = -3`,
not `-4`). The subtraction is performed in `i64` and **saturated** into the
`i32` range. Errors: `EmptyInput` if `input` is empty; `OutputLengthMismatch`
if `out.len() != input.len()`. Pinned by `dc_remove` (`PV-DC-*`).

### 9.2 Fixed-point FIR

`fir(input, coeffs, shift, out)` computes, for output index `n`,
`y[n] = (Σ coeffs[k] · input[n − k]) >> shift`, treating samples before the
start of the buffer as zero (causal, zero initial state). The accumulator is
`i64`; the right shift is **arithmetic** and applies round-half-up via a
`1 << (shift − 1)` bias when `shift >= 1`. Each output is **saturated** into
the `i32` range. Errors: `EmptyInput`; `EmptyKernel` if `coeffs` is empty;
`OutputLengthMismatch`; `InvalidShift` if `shift > 31`. To avoid `i64`
accumulator overflow, keep `coeffs.len() · max|coeffs| · max|input| < 2^63`.
`fir` is the deterministic convolution engine and makes **no** claim about
frequency response. Pinned by `fir` (`PV-FIR-*`).

### 9.3 DC blocker (stateful IIR)

`DcBlocker::step(x)` realises the first-order high-pass
`y[n] = x[n] − x[n−1] + R·y[n−1]`, with `R` a `Q15` coefficient in `0 < R < 1`
(default `32604` = `0.995`). It is evaluated in `i64` as
`y[n] = (((x[n] − x[n−1]) << 15) + R·y[n−1] + (1 << 14)) >> 15` and **saturated**
into the `i32` range. State is `(x[n−1], y[n−1])`; `reset()` zeroes it.
`with_r` rejects `R` outside `0 < R < 32768` with `InvalidCoefficient`.
`state_hash()` is FNV-1a 64 over `[R, x[n−1], y[n−1]]` as little-endian `i32`
(no pointers). Pinned by `dc_blocker` (`PV-DCB-*`), including the post-run
`state_hash`.

### 9.4 Biquad (stateful IIR notch / band-pass)

`Biquad::step(x)` realises a Direct-Form-I biquad with `Q15` coefficients and
`a0` normalised to `1`: `acc = b0·x[n] + b1·x[n−1] + b2·x[n−2] − a1·y[n−1] −
a2·y[n−2]` in `i64`, then `y[n] = (acc + (1 << 14)) >> 15`, **saturated** into
the `i32` range. State is `(x[n−1], x[n−2], y[n−1], y[n−2])`; `reset()` zeroes
it. `notch_coeffs` (50/60 Hz) and `bandpass_coeffs` (motor-intent / attention /
safety-wide) return tabulated coefficients for 250, 500, and 1000 Hz; the
`Disabled` selection is unity at any rate, and any other rate is rejected with
`UnsupportedSampleRate`. Coefficients were computed **offline** (RBJ cookbook)
and form an engineering-demonstrator design with **no** certified frequency
response (`docs/CLAIMS.md`, `docs/DSP_SPEC.md`). `state_hash()` is FNV-1a 64 over
`[b0, b1, b2, a1, a2, x[n−1], x[n−2], y[n−1], y[n−2]]` as little-endian `i32`.
Pinned by `biquad` (`PV-BIQ-*`), including the post-run `state_hash`.

## 10. Feature extraction (normative)

All features are **integer fixed-point**, computed per channel over a contiguous
sample buffer; outputs are bit-exact and carry no measured-quality claim.

- `isqrt(x)` is the floor integer square root; `log2_q16(x)` is base-2 log in
  `Q16` (16 fractional bits, `log2_q16(0) = 0` by convention). Pinned by `isqrt`
  (`PV-ISQRT-*`) and `log2_q16` (`PV-LOG2-*`).
- `variance` is the mean-removed population variance `Σ(x − mean)² / N` (`mean`
  truncated toward zero, `u128` accumulation); `log_variance_q16` is its `Q16`
  log; `rms` is `isqrt(variance)` (the standard deviation); `abs_mean` is
  `Σ|x| / N`; `zero_crossings` counts strict sign changes. Empty input is the
  error `EmptyInput`. Pinned by `feature` (`PV-FEAT-*`).

`FeatureVector<D>` remains a legacy `f32` interop container and is **not** part
of any conformance claim.

## 11. Classifier inference (normative)

The classifier evaluates fixed decision rules over a feature vector given
**caller-supplied** model parameters; there is no trained model. All arithmetic
is integer (`i128` accumulation, saturation), so a decision is a deterministic
function of (parameters, input).

- `distance_sq(feature, mean)` is the saturating squared Euclidean distance;
  length mismatch is `DimensionMismatch`.
- `classify_mdm(feature, class_means, abstain_below_permille)` returns the
  nearest class as a `ClassifierDecision`; confidence is the margin
  `1000·(d₂ − d₁)/(d₂ + d₁)` in permille, a tie gives `0`, and a confidence below
  the threshold yields `NoIntent`. Empty class set is `EmptyClassSet`.
- `lda_score(feature, weights, bias)` is the saturated linear score
  `bias + Σ weights·feature`; `classify_lda_binary(…, abstain_band)` decides class
  `1` for a positive score, `0` for negative, with a dead-band that yields
  `NoIntent` and a negative band rejected as `InvalidThreshold`.

Pinned by `classify_mdm` (`PV-MDM-*`) and `classify_lda` (`PV-LDA-*`).

## 12. Calibration (normative)

Calibration transforms are **integer/fixed-point** and pure functions of their
declared inputs; no accuracy, transfer, or convergence property is claimed.

- `covariance` is the mean-removed channel covariance
  `cov[i][j] = Σ(xᵢ − x̄ᵢ)(xⱼ − x̄ⱼ) / N`; dimension/length errors are
  `DimensionMismatch` / `EmptyInput` / `SampleLengthMismatch`. Pinned by
  `covariance` (`PV-COV-*`).
- `SessionMean` is the running mean of covariances; `drift_update` is the
  in-place `Q15` exponential update, with α outside `0 ..= 1` rejected as
  `InvalidCoefficient`.
- `whiten_cholesky(R)` returns the `Q16` Cholesky whitener `W = L⁻¹` with
  `W R Wᵀ = I` (the reference mapped to identity), or `None` if `R` is not
  positive-definite; `align(W, cov)` applies it as `W cov Wᵀ` in `Q16`. Both the
  whitener and the exact `align(W, R)` (the `Q16` identity to fixed-point error)
  are pinned by `whiten_cholesky` (`PV-WHIT-*`). The symmetric `R^{-1/2}`
  Euclidean Alignment form is deferred (`docs/CALIBRATION.md`).
- `ZeroCalib` is the typed skeleton that accumulates covariances and finalizes a
  reference whitener; it is structural only.
